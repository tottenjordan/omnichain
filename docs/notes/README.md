# Session Notes — Index

Topic-organized notes worth carrying across sessions. Each linked file covers a
single topic. Keep this index under 200 lines — it's a directory of pointers,
not a place for content.

## What belongs here

Save a note only if it will help a future session **and** isn't already
recoverable from the repo (git history, `CLAUDE.md`, existing docs). Capture
non-obvious insights you'd otherwise rediscover the hard way.

## Conventions

- One topic per file; link related topics between files.
- Check for an existing note before creating a new one — update, don't duplicate.
- Delete notes that turn out wrong or stale.
- Notes reflect what was true when written. Re-verify any named file/flag before
  acting on it.
